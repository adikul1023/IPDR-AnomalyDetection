# CyberShield A→B Mapping MVP

Fast, demo-ready tool for mapping A-party to B-party from IPDR logs, inferring encrypted links (VPN/E2E) via metadata correlation, and flagging anomalies with ML.

## Quickstart
```bash
python -m venv .venv && source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -r requirements.txt
streamlit run app.py
```
Upload a CSV (schema below) or use the provided sample.

### Sample data
- `data/sample_ipdr.csv` – main demo dataset
- `data/sample_ipdr_negative.csv` – generated by `python scripts/generate_synth_ipdr_negative.py`
- `data/sample_small.csv` – tiny 5-row quick demo
- `data/sample_ipdr_month.csv` – month-long synthetic dataset with diurnal patterns (`python scripts/generate_synth_ipdr_month.py`)

### Exports
- Direct edges: `edges.csv`
- Inferred pairs: `inferred_pairs.csv`
- Anomalies: `anomalies.csv`
- Graph HTML: `graph.html`

## IPDR CSV Schema
`timestamp, src_ip, src_port, dst_ip, dst_port, protocol, bytes_sent, bytes_received, duration_ms, session_id`

- `timestamp`: ISO8601 (UTC) string
- `session_id`: unique per flow/session (string or int)

## Modules
- `abmap/core.py`: parsing, direct A→B mapping, graph utilities
- `abmap/correlation.py`: VPN/E2E correlation using time overlap + traffic similarity
- `abmap/ml.py`: IsolationForest-based anomaly detection
- `abmap/threat.py`: lightweight threat enrichment (stub list + hooks)

## Notes
- Works fully offline for demo. Replace stubs with real feeds later.
- Keep datasets small (≤50k rows) for a smooth Streamlit demo.
